## 🔄 完整业务流程

### 场景1：模特查看作品并预约
```
1. 打开小程序 → 看到首页（摄影师名片+作品）
2. 浏览作品 → 点击喜欢的作品查看详情
3. 点击"立即预约" → 填写预约表单
4. 提交预约 → 收到"预约成功"提示
5. 切换到"预约"Tab → 在"我发出的"查看预约状态
6. 等待摄影师确认 → 状态变为"已确认"
7. 拍摄完成 → 状态变为"已完成"
```

### 场景2：摄影师管理预约
```
1. 打开小程序 → 切换到"预约"Tab
2. 查看"我收到的" → 看到新预约提醒
3. 点击"确认预约" → 模特收到确认通知
4. 拍摄当天 → 点击"完成拍摄"
5. 预约状态变为"已完成"
```

## 💡 关键设计点

### 为什么首页只展示"光影诗人"的作品？
- 这是**你的专属小程序**
- 模特打开后只能看到你的作品
- 不会被其他摄影师干扰
- 专注建立你的个人品牌

### 为什么保留"我发出的"？
- 模特需要**追踪自己的预约状态**
- 知道预约是待确认/已确认/已完成
- 完整的用户体验闭环

### 双Tab切换的意义
- **我发出的** = 模特视角
- **我收到的** = 摄影师视角
- 一个人既是模特又是摄影师时，可以切换查看
