# 📸 摄影师作品集平台 V3.0 - 统一架构设计

## 🎯 核心洞察

**每个人都是潜在的创作者** - 模特可以是摄影师，摄影师也可以是模特。不需要严格区分两端，而是提供一个统一的平台，让用户根据需要自由切换视角。

## 🏗️ 新架构：4个核心Tab

```
┌─────────────────────────────────────────────────────┐
│                    底部导航栏                         │
├─────────┬─────────┬─────────┬─────────────────────┤
│  发现   │  作品   │  预约   │       我的          │
│Discover │Portfolio│ Booking │       Me            │
└─────────┴─────────┴─────────┴─────────────────────┘
```

### 设计理念
- **统一入口**: 所有用户看到同样的4个Tab
- **动态功能**: 根据用户是否开启摄影师模式，动态显示功能
- **角色融合**: "我关注的"和"关注我的"都在一个页面
- **预约双向**: "我发出的预约"和"我收到的预约"统一管理

## 🔄 预约流程重构

### 场景1: 我是模特，想找摄影师

```
发现页 → 浏览作品 → 点击详情 → [预约拍摄]
              ↓
        填写预约表单
              ↓
        选择：期望时间、拍摄类型、备注
              ↓
        提交 → 等待摄影师确认
              ↓
        在"预约Tab-我发出的"查看状态
```

### 场景2: 我是摄影师，有人找我拍

```
收到推送通知
      ↓
打开"预约Tab-我收到的"
      ↓
查看待确认列表
      ↓
点击进入详情
      ↓
[确认预约] / [协商时间] / [婉拒]
      ↓
确认后进入"已确认"列表
      ↓
拍摄当天标记"拍摄中"
      ↓
后期完成标记"已完成"
```

### 场景3: 我既是模特又是摄影师

```
用户A是摄影师，也在平台上找其他摄影师拍摄

预约Tab显示：
┌────────────────────────────────┐
│ 📸 我收到的预约                │
│ 待确认 (2)                     │
│ • 用户X 想拍个人写真           │
│ • 用户Y 想拍情侣照             │
├────────────────────────────────┤
│ 🎯 我发出的预约                │
│ 已确认 (1)                     │
│ • 摄影师Z 确认2月20日拍摄      │
└────────────────────────────────┘
```

## 📊 数据模型调整

### 用户表 (Users) - 简化

```typescript
interface User {
  _id: string;              // openid
  
  // 基本信息
  nickName: string;
  avatarUrl: string;
  bio: string;              // 个人简介
  
  // 角色开关
  isPhotographer: boolean;  // 是否开启摄影师模式
  
  // 统计数据（统一）
  stats: {
    photos: number;         // 作品数
    likes: number;          // 获得收藏
    bookingsSent: number;   // 发出预约数
    bookingsReceived: number; // 收到预约数
  };
  
  // 摄影师资料（仅在isPhotographer=true时显示）
  photographerProfile?: {
    specialties: string[];  // 擅长风格
    equipment: string[];    // 设备
    priceRange: { min: number; max: number };
    contactWechat: string;
    qrCode: string;
  };
  
  createdAt: Date;
  updatedAt: Date;
}
```

### 预约表 (Bookings) - 优化

```typescript
interface Booking {
  _id: string;
  
  // 双方信息
  fromUserId: string;       // 发起者（模特）
  toUserId: string;         // 接收者（摄影师）
  
  // 状态（简化但完整）
  status: 'pending' |      // 待确认
          'confirmed' |     // 已确认
          'shooting' |      // 拍摄中
          'completed' |     // 已完成
          'cancelled';      // 已取消
  
  // 内容
  selectedPhotos: string[];
  preferredDates: string[];
  shootingType: string;
  notes: string;
  
  // 时间戳
  createdAt: Date;
  confirmedAt?: Date;
  completedAt?: Date;
}
```

**关键**: 不再需要区分"模特预约表"和"摄影师预约表"，统一用 `fromUserId` 和 `toUserId` 表示双向关系。

## ✅ 总结

### V3.0 核心改变

1. **从"双端"到"统一"**: 4个Tab，所有人用同一套界面
2. **角色灵活切换**: "我的"页面开关摄影师模式
3. **预约双向管理**: "我发出的"和"我收到的"在同一个页面
4. **人人都是创作者**: 作品管理对所有用户开放
5. **简化但不简陋**: 功能更集中，体验更流畅

### 优势

- ✅ 降低认知成本，不用记两套界面
- ✅ 支持身份重叠的用户（既是模特又是摄影师）
- ✅ 功能更聚合，不需要到处找
- ✅ 未来扩展更容易（比如添加"造型师"、"场地"等角色）
