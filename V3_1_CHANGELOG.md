### ✅ 6. 预约中心状态筛选
**原有设计**：我发出的和我收到的分开显示，但没有状态筛选
**修改**：在「我发出的」和「我收到的」Tab下方，增加状态筛选Tab

**7种状态**（横向滚动）：
1. **全部** - 显示所有预约
2. **待确认** - 等待摄影师/模特确认
3. **已确认** - 双方确认，等待拍摄
4. **拍摄中** - 正在进行拍摄
5. **待评价** - 拍摄完成，等待评价
6. **已完成** - 已评价，流程结束
7. **已取消** - 已取消的预约

**筛选逻辑**：
- 点击状态标签，只显示该状态的预约卡片
- 每个预约卡片标记data-status属性
- 状态标签带badge颜色区分

**状态流转**：
```
待确认 → 已确认 → 拍摄中 → 待评价 → 已完成
  ↓
已取消
```

### ✅ 10. 摄影师模式说明 + 删除支付设置

#### 摄影师模式存在的意义：

**为什么要区分？**
```
用户A：我只是想找摄影师拍照（纯模特需求）
用户B：我想找摄影师拍照，同时自己也接单（模特+摄影师）
用户C：我只接拍摄单，不发需求（纯摄影师）
```

**摄影师模式 = 开关控制**
- **关闭状态**：你是普通用户，只能发起预约（找别人拍）
- **开启状态**：你是摄影师，可以接收预约（别人找你拍）

**实际应用场景**：
1. **纯模特**：关闭摄影师模式，界面清爽，只看预约发出状态
2. **摄影爱好者**：平时关闭，偶尔开启接单
3. **专业摄影师**：长期开启，管理多个客户预约

**好处**：
- 一个App满足两种角色需求
- 不会错过接单机会
- 灵活切换，按需使用
- 数据显示：本月收到X个预约（成就感）

#### 删除支付设置的原因：

**为什么不加支付功能？**

1. **信任问题**：陌生人之间直接付款风险高
2. **纠纷处理**：拍摄不满意退款难处理
3. **平台责任**：涉及资金需要支付牌照和资质
4. **用户习惯**：摄影约拍通常是线下交易或微信转账
5. **复杂度**：增加支付会大幅提高开发和维护成本

**替代方案**：
- 线下支付（拍摄完成后现金/微信转账）
- 双方协商定价（平台不干预价格）
- 通过微信号联系后私下交易

**这样设计的好处**：
- 平台专注匹配和展示
- 用户有充分的自主权
- 避免资金纠纷影响平台声誉
- 开发周期短，快速上线
